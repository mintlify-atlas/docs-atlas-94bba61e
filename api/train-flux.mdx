## Overview

The `train_flux.py` script trains Flux Dev models using the MaxDiffusion framework. Flux training has been tested on TPU v5p with support for flash attention and bfloat16 precision.

## Command-line usage

```bash
python src/maxdiffusion/train_flux.py src/maxdiffusion/configs/base_flux_dev.yml [OPTIONS]
```

### Basic example

```bash
python src/maxdiffusion/train_flux.py \
  src/maxdiffusion/configs/base_flux_dev.yml \
  run_name="test-flux-train" \
  output_dir="gs://<your-gcs-bucket>/" \
  save_final_checkpoint=True \
  jax_cache_dir="/tmp/jax_cache"
```

## Performance

Expected results on 1024 x 1024 images with flash attention and bfloat16:

| Model | Accelerator | Sharding Strategy | Per Device Batch Size | Global Batch Size | Step Time (secs) |
| --- | --- | --- | --- | --- | --- |
| Flux-dev | v5p-8 | DDP | 1 | 4 | 1.31 |

## Configuration parameters

### Run configuration

<ParamField path="run_name" type="string" required>
  Name for this training run. Used for organizing outputs and metrics.
</ParamField>

<ParamField path="base_output_directory" type="string">
  GCS bucket path for outputs (e.g., `gs://my-bucket/`). Checkpoints and metrics will be saved here.
</ParamField>

<ParamField path="output_dir" type="string" default="sdxl-model-finetuned">
  Local or GCS directory for model outputs.
</ParamField>

<ParamField path="save_final_checkpoint" type="boolean" default={false}>
  Save the final checkpoint after training completes.
</ParamField>

### Model configuration

<ParamField path="pretrained_model_name_or_path" type="string" default="black-forest-labs/FLUX.1-dev">
  HuggingFace model identifier or local path to pretrained Flux model.
</ParamField>

<ParamField path="clip_model_name_or_path" type="string" default="ariG23498/clip-vit-large-patch14-text-flax">
  HuggingFace model identifier for CLIP text encoder.
</ParamField>

<ParamField path="t5xxl_model_name_or_path" type="string" default="ariG23498/t5-v1-1-xxl-flax">
  HuggingFace model identifier for T5-XXL text encoder.
</ParamField>

<ParamField path="flux_name" type="string" default="flux-dev">
  Flux model variant name. Options: `flux-dev`, `flux-schnell`.
</ParamField>

<ParamField path="unet_checkpoint" type="string" default="">
  Path to a specific transformer checkpoint to load.
</ParamField>

<ParamField path="revision" type="string" default="refs/pr/95">
  Model revision/branch to use from HuggingFace.
</ParamField>

<ParamField path="weights_dtype" type="string" default="bfloat16">
  Data type for model weights. Options: `float32`, `bfloat16`.
</ParamField>

<ParamField path="activations_dtype" type="string" default="bfloat16">
  Data type for layer activations. Options: `float32`, `bfloat16`.
</ParamField>

<ParamField path="precision" type="string" default="DEFAULT">
  JAX precision for matmul and conv operations. Options: `DEFAULT`, `HIGH`, `HIGHEST`.
</ParamField>

<ParamField path="from_pt" type="boolean" default={true}>
  Load weights from PyTorch format.
</ParamField>

<ParamField path="train_new_flux" type="boolean" default={false}>
  If true, randomly initialize Flux weights to train from scratch. Otherwise, load from pretrained model.
</ParamField>

### Flux-specific parameters

<ParamField path="max_sequence_length" type="number" default={512}>
  Maximum text sequence length for T5 encoder.
</ParamField>

<ParamField path="time_shift" type="boolean" default={true}>
  Enable time shifting for Flux flow matching.
</ParamField>

<ParamField path="base_shift" type="number" default={0.5}>
  Base shift parameter for Flux.
</ParamField>

<ParamField path="max_shift" type="number" default={1.15}>
  Maximum shift parameter for Flux.
</ParamField>

<ParamField path="offload_encoders" type="boolean" default={true}>
  Offload T5 encoder after text encoding to save memory.
</ParamField>

### Attention configuration

<ParamField path="attention" type="string" default="flash">
  Attention mechanism to use. Options: `dot_product`, `flash`, `cudnn_flash_te`.
</ParamField>

<ParamField path="split_head_dim" type="boolean" default={true}>
  Whether to split attention head dimensions for sharding.
</ParamField>

<ParamField path="attention_sharding_uniform" type="boolean" default={true}>
  Use uniform sequence sharding for both self and cross attention.
</ParamField>

<ParamField path="mask_padding_tokens" type="boolean" default={true}>
  Pass segment IDs to attention to avoid attending to padding tokens. Improves quality when padding is significant.
</ParamField>

<ParamField path="flash_block_sizes" type="object" default={{}}>
  Custom block sizes for flash attention. On v6e (Trillium), use larger blocks:
  ```yaml
  flash_block_sizes: {
    "block_q" : 1536,
    "block_kv_compute" : 1536,
    "block_kv" : 1536,
    "block_q_dkv" : 1536,
    "block_kv_dkv" : 1536,
    "block_kv_dkv_compute" : 1536,
    "block_q_dq" : 1536,
    "block_kv_dq" : 1536
  }
  ```
</ParamField>

<ParamField path="learning_rate" type="number" default={1e-5}>
  Learning rate for the optimizer.
</ParamField>

<ParamField path="scale_lr" type="boolean" default={false}>

</ParamField>

<ParamField path="max_train_steps" type="number" default={1500}>

</ParamField>

<ParamField path="per_device_batch_size" type="number" default={1}>
  Batch size per device.
</ParamField>

<ParamField path="warmup_steps_fraction" type="number" default={0.1}>
  Fraction of total steps to use for learning rate warmup.
</ParamField>

<ParamField path="seed" type="number" default={0}>
  Random seed for reproducibility.
</ParamField>

### Optimizer parameters

<ParamField path="adam_b1" type="number" default={0.9}>
  Exponential decay rate for first moment estimates.
</ParamField>

<ParamField path="adam_b2" type="number" default={0.999}>
  Exponential decay rate for second moment estimates.
</ParamField>

<ParamField path="adam_eps" type="number" default={1e-8}>
  Small constant for numerical stability.
</ParamField>

<ParamField path="adam_weight_decay" type="number" default={0}>
  Weight decay coefficient for AdamW optimizer.
</ParamField>

<ParamField path="max_grad_norm" type="number" default={1.0}>
  Maximum gradient norm for gradient clipping.
</ParamField>

### Loss and noise schedule

<ParamField path="snr_gamma" type="number" default={-1.0}>
  SNR-weighted loss gamma parameter. Set to -1.0 to disable.
</ParamField>

<ParamField path="timestep_bias" type="object">
  Configuration for biasing timestep sampling during training.
  
  - `strategy`: Bias strategy. Options: `none`, `earlier`, `later`, `range`
  - `multiplier`: Bias multiplier (2.0 doubles weight, 0.5 halves it)
  - `begin`: Start timestep for range strategy
  - `end`: End timestep for range strategy
  - `portion`: Fraction of timesteps to bias
</ParamField>

<ParamField path="diffusion_scheduler_config" type="object">
  Override parameters for the diffusion scheduler.
  
  - `_class_name`: Scheduler class name (default: `FlaxEulerDiscreteScheduler`)
  - `prediction_type`: Prediction type (default: `epsilon`)
  - `rescale_zero_terminal_snr`: Whether to rescale zero terminal SNR
  - `timestep_spacing`: Timestep spacing strategy (default: `trailing`)
</ParamField>

## Expected outputs

### Checkpoints

When `save_final_checkpoint` is true or `checkpoint_every` is set, checkpoints are saved to:
```
{base_output_directory}/{run_name}/checkpoints/
```

Checkpoints include:
- Flux transformer weights
- Optimizer state
- Training step information

### Metrics

Training metrics are saved to:
```
{base_output_directory}/{run_name}/metrics/
```

Metrics include:
- Training loss
- Learning rate
- TFLOPS per device
- Step time

### Tensorboard logs

Tensorboard logs are written to:
```
{base_output_directory}/{run_name}/tensorboard/
```

View logs with:
```bash
tensorboard --logdir=gs://your-bucket/{run_name}/tensorboard/
```
